# Диагностика бота на Render

## Что проверить, если бот не работает:

### 1. Переменные окружения на Render
Убедитесь, что в настройках вашего сервиса на Render установлены:
- `BOT_TOKEN` - токен вашего Telegram бота
- `WEBHOOK_URL` или `RENDER_EXTERNAL_URL` - URL вашего приложения на Render (например: https://your-app-name.onrender.com)

### 2. Проверка статуса бота
После деплоя перейдите по адресу:
- `https://your-app-name.onrender.com/status` - покажет состояние бота и webhook

### 3. Логи на Render
Проверьте логи в панели Render:
- Должно быть сообщение "Webhook установлен успешно"
- Должна быть информация о боте (Bot info)
- Не должно быть ошибок при установке webhook

### 4. Возможные проблемы:

#### Проблема: Webhook не устанавливается
**Решение**: Проверьте переменную окружения WEBHOOK_URL или RENDER_EXTERNAL_URL

#### Проблема: Бот не отвечает на сообщения
**Решение**: 
- Проверьте endpoint /status
- Убедитесь, что webhook URL правильный
- Проверьте логи на наличие ошибок

#### Проблема: 502 Bad Gateway
**Решение**: 
- Приложение может долго запускаться на бесплатном плане
- Подождите 1-2 минуты после деплоя

### 5. Команды для ручной проверки webhook через BotFather:
Если нужно сбросить webhook вручную, отправьте POST запрос:
```
https://api.telegram.org/bot{YOUR_BOT_TOKEN}/setWebhook?url=https://your-app-name.onrender.com/webhook
```

### 6. Тестирование локально:
Для локального тестирования используйте:
```bash
npm run dev
```
Это запустит index.js с polling вместо webhook.
